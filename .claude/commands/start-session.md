# Start Session Protocol - PC-Inspector

Execute these steps to initialize a development session:

## 1. Context Review
- Read `docs/MASTER_HANDOFF_INDEX.md` for session history
- Read the most recent handoff from `docs/handoffs/`
- Check `docs/BACKLOG.md` for pending features

## 2. Status Check
- Run `git status` to check for uncommitted changes
- Check if venv exists and dependencies are installed
- Verify database exists at `data/system.db`

## 3. Environment Verification
- Verify Python 3.12+ is installed
- Verify venv is set up
- Test one-click startup scripts (start.bat, start.ps1, start.sh)

## 4. Session Summary
Provide:
- **Last session focus**: What was accomplished
- **Current status**: What's working, what needs fixing
- **Next priorities**: From backlog or previous session
- **Known issues**: Any bugs or blockers

## Output
After completing the protocol, present:
1. Quick status summary (1-2 sentences)
2. Recommended next actions
3. Any issues requiring immediate attention
