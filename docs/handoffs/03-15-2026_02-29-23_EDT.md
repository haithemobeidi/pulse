## Technical Notes
- The H3 learning patterns were adapted to SQLite (no Neo4j) — graph traversals replaced by SQL JOINs
- Confidence decay formula: `effective = stored * 0.5^(days_since_activity / 30)`
- Similarity scoring: `score = cosine_similarity * 0.7 + decayed_confidence * 0.3`
- SSE route is on the app directly (not blueprint) to avoid Flask's catch-all static route conflict
- The `ui-redesign` branch changes (blueprint refactor) were merged into this work since the H3 code depends on the services layer
- Background scheduler uses threading.Timer (no APScheduler dependency), checks hourly
