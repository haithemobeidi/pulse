## Technical Notes
- Collectors now run in parallel via `concurrent.futures.ThreadPoolExecutor` - hardware, monitors, and reliability all execute simultaneously. This cut scan time significantly
- The reliability collector's PowerShell query was broken because `Get-CimInstance` returns `DateTime` objects, not WMI datetime strings. The fix uses `(Get-Date).AddDays(-N)` for comparison instead of formatted strings
- Pulse Manager had a race condition: `stop_server()` sets `self.server_process = None`, but the background stdout-reading thread then tries `self.server_process.wait()` → NoneType error. Fixed by null-checking before access
- The `.ts` files in frontend are legacy from the initial build - the active files are all `.js`. Should be cleaned up
