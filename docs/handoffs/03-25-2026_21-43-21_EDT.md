## Technical Notes

### Metabolism Architecture
- Runs asynchronously via `digest_session_async()` in a daemon thread using the existing db connection (check_same_thread=False)
- Uses qwen3:30b-a3b (batch worker) — fast, free, keeps primary model free for user interaction
- Simplified extraction prompt is critical: complex system prompts cause qwen3 MoE to put everything in the thinking field, returning empty content
- JSON parser has 3-tier fallback: direct parse → brace-matched extraction → regex fallback
- Thinking field fallback: if content is empty but thinking has data, uses thinking content for parsing

### Qwen3 Thinking Mode Gotcha
The biggest debugging challenge this session: qwen3 MoE models (30b-a3b) have a "thinking" mode that returns internal reasoning in a separate `thinking` field in the Ollama API response. With complex system prompts, the model sometimes puts ALL output (including the desired JSON) into the thinking field, leaving `content` empty. The fix involves:
1. Using `/no_think` token prepended to user message
2. Keeping the system prompt short and direct
3. Falling back to the thinking field when content is empty
4. Aggressive JSON extraction from verbose text (brace-counting + regex)
