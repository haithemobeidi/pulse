## Technical Notes
- DuckDuckGo HTML API (`html.duckduckgo.com/html/`) is more reliable than the `duckduckgo_search` Python library which gets blocked
- DDG returns 202 status when rate-limited; retry with exponential delay handles this
- Session memory uses SQLite UPSERT (`ON CONFLICT DO UPDATE`) for atomic key updates
- Web search queries are hardware-aware: includes GPU model + driver version for targeted results (e.g., "RTX 5090 driver 595.76 known issues")
- The 2-agent architecture decision: 1 Master Troubleshooter (with dynamic context modules instead of separate hardware agents) + 1 Builder Agent (Claude with tool_use for code editing)
- Phase ordering was revised after architecture audit: Memory first (prerequisite for everything), then Routing (cheaper than full supervisor), then Builder, then Resolution Paths
