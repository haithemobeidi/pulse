## Technical Notes
- All collectors now run in parallel via `concurrent.futures.ThreadPoolExecutor`
- The reliability collector was broken because `Get-CimInstance` returns `DateTime` objects, not WMI datetime strings — fixed by using `(Get-Date).AddDays(-N)` instead of formatted string comparison
- Pulse Manager had a threading race condition: `stop_server()` sets `server_process = None` while the background stdout-reading thread still references it. Fixed with `_intentional_stop` flag
- Browser bfcache can restore DOM state from previous sessions, making hidden elements visible. Using `style.display:none` (inline style) instead of CSS classes prevents this
- Server startup auto-collect runs in a daemon thread with 2s delay to ensure Flask is ready
- The `logger` variable must be defined at module level in `app.py` — it was previously only in a local function scope, causing NameError on startup
