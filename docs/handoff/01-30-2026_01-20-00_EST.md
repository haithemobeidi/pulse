## Accomplishments This Session

### 1. Architecture Pivot: Flask MVP (Major Win)
- **Problem**: User experiencing Pydantic/Rust compilation errors, complex multi-server setup
- **Solution**: Replaced FastAPI + separate HTTP server with single Flask server
- **Benefit**: One-command startup, no build step, simpler deployment
- **Result**: `python -m backend.app` on port 5000 handles both API and static files

### 2. Resolved Threading Issues
- **Problem**: SQLite threading error - "objects created in a thread can only be used in that same thread"
- **Solution**: Added `check_same_thread=False` to sqlite3.connect()
- **Impact**: Fixed all 500 errors when Flask handled requests in different threads

### 3. Migrated Data Collection to Python Libraries
- **Problem**: PowerShell queries returning empty data, GPUtil broken (distutils missing in Python 3.14)
- **Solution**: Switched to proven libraries:
  - `psutil` for CPU/Memory (already installed, highly reliable)
  - `wmi` for GPU and monitor detection
  - Eliminated PowerShell subprocess calls entirely
- **Result**: Clean, working data collection

### 4. Fixed Frontend Display Issues
- **Problem 1**: TypeScript files loading in browser causing MIME type errors
  - **Solution**: Converted all TypeScript to plain JavaScript (no compilation needed)
- **Problem 2**: API client pointing to wrong port (8000 vs 5000)
  - **Solution**: Updated to localhost:5000
- **Problem 3**: Dashboard field names not matching API response
  - **Solution**: Corrected all field names (cpu_count→physical_cores, etc.)

### 5. GPU Detection Implementation
- **Problem**: GPUtil unavailable (Python 3.14 incompatibility)
- **Solution**: Implemented WMI-based GPU detection with fallback logic
- **Status**: RTX 5090 now detected ✅
- **Known Issue**: VRAM showing "Unknown" (WMI returns negative/invalid values)

### 6. Monitor Detection Implementation
- **Problem**: Initially detected 0 monitors, then generic names
- **Solution**: Implemented 4-method fallback strategy:
  1. WMI PnP devices (primary)
  2. WMI desktop monitors
  3. WMI video controllers
  4. Windows registry (HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Enum\DISPLAY)
- **Status**: 2 monitors detected ✅
- **Known Issue**: Generic names ("Default Monitor", "Generic PnP Monitor") - not friendly names

### 7. Future Features Documentation
- **Created**: `/docs/FUTURE_FEATURES.md`
- **Covers**: Phase 2-4 roadmap including:
  - Real-time per-core CPU utilization capture
  - Self-learning anomaly detection (2σ baseline)
  - Auto-issue creation when anomalies detected
  - Correlation analysis (metrics → issue patterns)
  - Driver version impact tracking
